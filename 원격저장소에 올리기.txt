수정사항이 있을 때
-git add . 
-git commit -m "Add files" 
-git push origin master

원격 저장소와 연결이 다른 디렉터리에 되어 있다면 새로운 디렉터리에서 원격 저장소로 연결이 잘 되므로 기존의 연결을 끊어야 한다.
-git remote remove origin
새로운 디렉터리에 아래 명령어 실행
-git remote add origin https://github.com/practicaluser/git-practice
파일을 업데이트
-git push origin master

깃허브 처음 연결할 때(원격저장소도 처음 만들어진 경우)
git init && git add . && git commit -m "Add files" 
git remote add origin https://github.com/practicaluser/devops-blog.git
git push origin master

잔디심을려면 
-git 저장소에 파일들이 push되었어도, push를 실행한 로컬의 이메일 정보가 git의 계정과 일치해야 잔디가 심어진다
-계정 정보는 로컬, 글로벌이 있는데 글로벌 정보만 바꾸어 주어도 해결이 된다.
-git config --global user.name "practicaluser"
-git config --global user.email "dlcnsdncjswo@naver.com"
-git config user.name
-git config user.email
-git config --unset user.name(해제시)

저장소
-등록된 로컬저장소 이름 확인
--git remote

-리모트 저장소의 이름을 변경(origin을 cjswo로 변경)
--git remote rename origin cjswo

브랜치
-현재 작업하고 있는 브랜치 확인
--git branch

-브랜치 생성
--git branch 브랜치이름

-브랜치 전환하기
--git checkout <branch>

-브랜치 삭제
--git branch -d 브랜치이름

-원격 저장소에 있는 브랜치 가져오기
--먼저 git remote를 갱신하기 : git remote update
--원격 저장소의 branch 확인 : git branch -r
--만약 원격 저장소의 feature/create-meeting branch를 가져오고 싶을 때
---git checkout -t origin/feature/create-meeting
